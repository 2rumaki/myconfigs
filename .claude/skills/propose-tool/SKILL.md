# /propose-tool

ツール名またはリポジトリURLを受け取り、調査結果をGitHub issueとして起票するスキル。

引数: $ARGUMENTS (ツール名またはリポジトリURL)

## 手順

1. 以下の issue テンプレートの構造を把握する

2. WebSearch および WebFetch を使って「$ARGUMENTS」について以下の情報を調査する:
   - ツールの公式リポジトリ・公式サイト
   - 概要・主な機能
   - インストール方法
   - 基本的な使い方
   - 管理者・メンテナー情報（個人 or 組織、主要コントリビューター）
   - メンテナンス状況（最終リリース日、リリース頻度、コントリビューター数）
   - 最新バージョン
   - 依存関係
   - 類似ツール（2〜3個）の情報

3. 類似ツールについても同様に Star 数、ライセンス、最終リリース日、リリース頻度、コントリビューター数、最新バージョン、依存関係を調査する

4. 以下のテンプレートに沿って issue 本文を作成する

5. 作成内容をユーザーに提示し、確認を得る

6. `gh issue create` で issue を起票する
   - タイトル形式: `[ツール検討] {ツール名} - {ツールの簡潔な説明}`
   - ラベルは付与しない

## 注意事項
- 調査結果は可能な限り公式情報・GitHubリポジトリの実データに基づくこと
- Star 数やリリース日などの数値は調査時点の値を記載し、推測で埋めないこと
- 情報が取得できなかった項目は「不明」と記載すること
- issue 作成前に、必ず作成内容をユーザーに確認すること
