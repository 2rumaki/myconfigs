# /propose-tool

ツール名またはリポジトリURLを受け取り、調査結果をGitHub issueとして起票するスキル。

引数: $ARGUMENTS (ツール名またはリポジトリURL)

## 手順

1. 以下の issue テンプレートの構造を把握する

2. WebSearch および WebFetch を使って「$ARGUMENTS」について、issue テンプレートに記載のすべての項目を調査する。類似ツール（2〜3個）についても比較表の各項目を同様に調査する

3. テンプレートに沿って issue 本文を作成する

4. 作成内容をユーザーに提示し、確認を得る

5. `gh issue create` で issue を起票する
   - タイトル形式: `[ツール検討] {ツール名} - {ツールの簡潔な説明}`
   - ラベルは付与しない

## 注意事項
- 調査結果は可能な限り公式情報・GitHubリポジトリの実データに基づくこと
- Star 数やリリース日などの数値は調査時点の値を記載し、推測で埋めないこと
- 情報が取得できなかった項目は「不明」と記載すること
- issue 作成前に、必ず作成内容をユーザーに確認すること
